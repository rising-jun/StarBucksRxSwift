# Starbucks Clone App 기획 문서

## 1. 문서 목적

이 문서는 로그인 이전 구간만 구현하는 스타벅스 클론앱의 화면 기획 문서입니다.
목표는 실제 스타벅스 앱의 전체 기능을 복제하는 것이 아니라, 아래 범위 안에서 자연스러운 탐색 경험을 제공하는 것입니다.

- 홈 화면에서 시즌감 있는 프로모션과 대표 메뉴를 보여준다.
- 이벤트 화면에서 진행 중인 프로모션을 탐색할 수 있다.
- 메뉴 화면에서 카테고리별 메뉴를 둘러보고 상세 정보를 확인할 수 있다.

이번 단계에서는 학습용 구현을 우선하며, 로그인 이후 기능은 모두 제외합니다.

## 2. 서비스 범위

### 포함 기능

- 홈 메인 화면
- 이벤트 목록 화면
- 이벤트 상세 화면
- 메뉴 카테고리 화면
- 메뉴 목록 화면
- 메뉴 상세 화면

### 제외 기능

- 로그인 / 회원가입
- 결제 / 주문 / 장바구니
- 리워드 / 쿠폰 / 스탬프 적립
- 매장 찾기 / 지도 / 픽업
- 검색
- 찜 / 좋아요 / 최근 본 메뉴
- 푸시 알림 / 설정

## 3. 앱 구조 제안

학습용 범위에서는 하단 탭을 단순하게 유지합니다.

| 탭 | 설명 |
| --- | --- |
| `Home` | 프로모션과 추천 콘텐츠 중심 |
| `Event` | 이벤트 목록과 상세 확인 |
| `Menu` | 카테고리 및 메뉴 탐색 |

탭별 진입 구조는 아래와 같습니다.

```text
Home
 ├─ Event Detail
 └─ Menu Detail

Event
 └─ Event Detail

Menu
 ├─ Category Selector
 ├─ Menu List
 └─ Menu Detail
```

## 4. 사용자 시나리오

### 시나리오 1. 홈에서 시즌 메뉴를 보고 상세로 진입

1. 사용자가 앱을 실행한다.
2. 홈 상단 배너에서 시즌 음료를 본다.
3. 배너를 탭한다.
4. 메뉴 상세 화면으로 이동한다.

### 시나리오 2. 이벤트 내용을 확인하고 관련 메뉴로 이동

1. 사용자가 홈 또는 이벤트 탭에서 이벤트를 선택한다.
2. 이벤트 상세 화면에서 기간, 안내 이미지, 설명을 본다.
3. 하단 CTA 버튼을 탭한다.
4. 메뉴 탭 또는 특정 카테고리로 이동한다.

### 시나리오 3. 메뉴를 둘러보다가 상세 확인

1. 사용자가 메뉴 탭으로 이동한다.
2. 카테고리를 선택한다.
3. 메뉴 목록을 스크롤하며 이미지, 이름, 가격을 확인한다.
4. 메뉴를 탭해 상세 화면으로 이동한다.

## 5. 화면 목록

| 화면 ID | 화면명 | 우선순위 | 비고 |
| --- | --- | --- | --- |
| `SCR-001` | 홈 메인 화면 | High | 앱 첫 화면 |
| `SCR-002` | 이벤트 목록 화면 | High | 이벤트 탭 루트 |
| `SCR-003` | 이벤트 상세 화면 | High | 홈/이벤트 공통 진입 |
| `SCR-004` | 메뉴 메인 화면 | High | 카테고리 + 목록 |
| `SCR-005` | 메뉴 상세 화면 | High | 홈/메뉴 공통 진입 |
| `SCR-006` | 공통 로딩/에러 화면 상태 | Medium | 각 화면 공통 정책 |

## 6. 화면 상세 기획

### SCR-001 홈 메인 화면

#### 화면 목적

- 앱의 첫인상을 만든다.
- 시즌성 콘텐츠와 대표 메뉴를 빠르게 노출한다.
- 이벤트와 메뉴 상세 화면으로 진입시키는 허브 역할을 한다.

#### 주요 구성 요소

| 영역 | 내용 |
| --- | --- |
| 상단 헤더 | 앱 타이틀 또는 심플한 브랜드 타이틀 |
| 메인 배너 | 시즌 음료 또는 대표 프로모션 1~3개 |
| 추천 메뉴 섹션 | 대표 메뉴 카드 리스트 |
| 이벤트 섹션 | 진행 중 이벤트 카드 또는 캐러셀 |
| 프로모션 배너 | 외부 링크 없이 내부 상세로 연결되는 배너 |

#### 반드시 보여줘야 하는 정보

- 시즌 배너 이미지
- 배너 제목 / 부제목
- 추천 메뉴 이미지, 이름, 가격
- 이벤트 썸네일, 제목, 상태 라벨

#### 사용자 액션

- 배너 탭
- 추천 메뉴 탭
- 이벤트 카드 탭
- 아래로 스크롤

#### 화면 상태

- 로딩: 스켈레톤 또는 `UIActivityIndicatorView`
- 성공: 섹션 노출
- 빈 상태: "현재 노출할 콘텐츠가 없습니다."
- 에러: 재시도 버튼 노출

#### 연결 API

- `GET /api/v1/home`

### SCR-002 이벤트 목록 화면

#### 화면 목적

- 현재 진행 중인 이벤트를 한 곳에서 탐색하게 한다.
- 이벤트별 제목, 기간, 썸네일을 빠르게 비교할 수 있게 한다.

#### 주요 구성 요소

| 영역 | 내용 |
| --- | --- |
| 화면 헤더 | `이벤트` 타이틀 |
| 상태 필터 | `진행중`, `예정`, `종료` |
| 이벤트 리스트 | 세로 리스트 형태의 이벤트 카드 |

#### 반드시 보여줘야 하는 정보

- 이벤트 썸네일 이미지
- 이벤트 제목
- 요약 문구
- 진행 상태 라벨
- 이벤트 기간

#### 사용자 액션

- 상태 필터 변경
- 이벤트 셀 선택
- 리스트 스크롤

#### 화면 상태

- 로딩: 리스트 스켈레톤
- 성공: 이벤트 리스트 노출
- 빈 상태: "해당 상태의 이벤트가 없습니다."
- 에러: 재시도 버튼 노출

#### 연결 API

- `GET /api/v1/events?status={status}`

### SCR-003 이벤트 상세 화면

#### 화면 목적

- 이벤트의 전체 내용을 읽을 수 있게 한다.
- 관련 메뉴나 카테고리로 이동시키는 전환 화면 역할을 한다.

#### 주요 구성 요소

| 영역 | 내용 |
| --- | --- |
| 상단 이미지 | 이벤트 대표 비주얼 |
| 제목 영역 | 제목, 요약 문구, 기간 |
| 본문 콘텐츠 | 이미지 + 텍스트 조합 |
| 하단 CTA | 관련 메뉴 보러가기 버튼 |

#### 반드시 보여줘야 하는 정보

- 이벤트 제목
- 기간
- 대표 이미지
- 상세 안내 텍스트 또는 상세 이미지
- CTA 버튼 텍스트

#### 사용자 액션

- 본문 스크롤
- CTA 버튼 탭
- 뒤로가기

#### 화면 상태

- 로딩: 상단 이미지 및 본문 플레이스홀더
- 성공: 상세 콘텐츠 노출
- 에러: "이벤트 정보를 불러오지 못했습니다."

#### 연결 API

- `GET /api/v1/events/{eventId}`

### SCR-004 메뉴 메인 화면

#### 화면 목적

- 메뉴 탐색의 시작점이 된다.
- 카테고리를 바꾸며 메뉴를 둘러보게 한다.

#### 주요 구성 요소

| 영역 | 내용 |
| --- | --- |
| 화면 헤더 | `메뉴` 타이틀 |
| 카테고리 탭 | 음료 / 푸드 / 상품 |
| 메뉴 리스트 | 이미지 중심 카드 또는 리스트 |
| 필터 영역 | 선택적으로 `NEW`, `BEST`, `HOT/ICED` |

#### 반드시 보여줘야 하는 정보

- 카테고리명
- 메뉴 썸네일
- 메뉴명
- 영문명
- 가격
- 배지 정보
- 품절 여부

#### 사용자 액션

- 카테고리 선택
- 필터 선택
- 메뉴 셀 선택
- 리스트 스크롤

#### 화면 상태

- 로딩: 카테고리 및 리스트 스켈레톤
- 성공: 메뉴 목록 노출
- 빈 상태: "조건에 맞는 메뉴가 없습니다."
- 에러: 재시도 버튼 노출

#### 연결 API

- `GET /api/v1/menu/categories`
- `GET /api/v1/menu/items?categoryId={categoryId}`

### SCR-005 메뉴 상세 화면

#### 화면 목적

- 메뉴를 충분히 설명해 사용자가 실제 앱처럼 탐색 경험을 느끼게 한다.
- 주문은 하지 않더라도 제품 정보를 구조적으로 보여준다.

#### 주요 구성 요소

| 영역 | 내용 |
| --- | --- |
| 상단 이미지 | 메뉴 대표 이미지 |
| 기본 정보 영역 | 메뉴명, 영문명, 설명, 가격 |
| 배지 영역 | `NEW`, `BEST`, 품절 |
| 영양 정보 영역 | 칼로리, 당류, 단백질, 카페인 |
| 알레르기 정보 | 원재료 / 알레르기 정보 |
| 추가 설명 | 제품 소개 문구 |

#### 반드시 보여줘야 하는 정보

- 메뉴 이미지
- 메뉴명 / 영문명
- 설명
- 가격
- 온도 타입
- 영양 정보
- 알레르기 정보

#### 사용자 액션

- 상세 스크롤
- 뒤로가기

#### 화면 상태

- 로딩: 대표 이미지와 텍스트 플레이스홀더
- 성공: 상세 노출
- 에러: "메뉴 정보를 불러오지 못했습니다."

#### 연결 API

- `GET /api/v1/menu/items/{menuId}`

### SCR-006 공통 상태 정책

#### 로딩

- 첫 진입에서는 전체 화면 스피너보다 섹션/리스트 단위 스켈레톤을 우선한다.
- API 재호출 시에는 기존 데이터가 있으면 전체 화면 블로킹을 피한다.

#### 빈 상태

- 콘텐츠가 없는 경우 단순 문구와 가벼운 보조 설명만 보여준다.
- 학습용 앱이므로 과도한 일러스트는 필수 아님.

#### 에러 상태

- 네트워크 실패와 데이터 없음은 분리해서 표현한다.
- 버튼 문구는 `다시 시도`로 통일한다.

## 7. 네비게이션 정책

### 기본 정책

- 각 탭은 `UINavigationController` 루트로 구성한다.
- 상세 화면은 push 전환을 사용한다.
- 탭 간 이동은 사용자가 명시적으로 탭을 선택하거나 CTA를 누른 경우만 발생한다.

### 화면 이동 규칙

| 출발 | 액션 | 도착 |
| --- | --- | --- |
| 홈 메인 | 시즌 배너 탭 | 메뉴 상세 |
| 홈 메인 | 이벤트 카드 탭 | 이벤트 상세 |
| 이벤트 목록 | 이벤트 셀 탭 | 이벤트 상세 |
| 이벤트 상세 | CTA 탭 | 메뉴 탭 또는 메뉴 카테고리 |
| 메뉴 메인 | 메뉴 셀 탭 | 메뉴 상세 |

## 8. UI 우선순위 제안

구현은 아래 순서가 효율적입니다.

1. 홈 메인 화면
2. 메뉴 메인 화면
3. 메뉴 상세 화면
4. 이벤트 목록 화면
5. 이벤트 상세 화면
6. 공통 로딩 / 에러 상태 정리

이 순서가 좋은 이유는 홈과 메뉴가 앱의 기본 뼈대를 만들고, 이후 이벤트 상세는 같은 패턴으로 확장하기 쉽기 때문입니다.

## 9. 디자인 톤 가이드

- 전반적으로 화이트 기반에 스타벅스 그린을 포인트 컬러로 사용
- 카드형 UI를 사용하되 과한 그림자보다는 깔끔한 분리감 우선
- 홈 배너는 이미지 비중을 높여 브랜드 무드를 전달
- 메뉴 화면은 가독성이 우선이므로 이미지와 텍스트 간 간격을 명확히 유지
- 이벤트 상세는 이미지 중심의 읽는 화면으로 구성

## 10. 개발 시 체크 포인트

- 홈은 섹션형 데이터 구조로 만들어 이후 구성 변경에 유연하게 대응
- 이벤트와 메뉴는 리스트 + 상세 패턴을 재사용 가능하게 설계
- API 실패 시 화면이 비지 않도록 상태 바인딩 구조를 먼저 정의
- 로그인 이후 기능이 없으므로 CTA는 주문 유도가 아니라 탐색 유도로 제한
